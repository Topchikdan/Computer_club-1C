## О проекте

Конфигурация объединяет несколько контуров автоматизации клуба: бронирование и эксплуатация залов, торговля и бар, работа с клиентами, кадровый учёт и аналитика.

Исходники выгружены в XML-формате штатным механизмом платформы (**Конфигурация → Выгрузить конфигурацию в файлы...**) и версионируются через git. Формат позволяет вести историю изменений и работает на базовых версиях платформы без хранилища конфигурации.

---

## Возможности

- Учёт залов, компьютеров и игр клуба
- Бронирование мест клиентами
- Продажа и закупка товаров и услуг, включая ассортимент бара
- Складской учёт остатков и списание товаров
- Ведение карточек клиентов с поддержкой VIP-скидок
- Базовый кадровый учёт: приём, увольнение, перемещения сотрудников
- Отчёты по выручке и состоянию склада

## Состав конфигурации

### Справочники

| Справочник | Назначение |
|---|---|
| `Залы` | Игровые залы клуба |
| `Компьютеры` | Учёт компьютеров клуба (привязка к залам, характеристики) |
| `Игры` | Каталог игр, доступных в клубе |
| `Клиенты` | Карточки клиентов, в т.ч. с признаком VIP |
| `ТоварыИУслуги` | Номенклатура товаров и услуг (включая ассортимент бара) |
| `ВидТоваровИУслуг` | Классификация товаров и услуг |
| `Сотрудники` | Карточки сотрудников клуба |
| `Должности` | Перечень должностей |
| `Отделы` | Структурные подразделения |
| `ПолСотрудника` | Справочник пола сотрудника |

### Документы

| Документ | Назначение |
|---|---|
| `Бронирование` | Оформление брони зала/компьютера клиентом |
| `ПродажаТоваров` | Продажа товаров и услуг клиенту |
| `ПокупкаТоваров` | Закупка товаров (поступление на склад) |
| `СписаниеТоваров` | Списание товаров со склада |
| `ПриемНаРаботу` | Оформление приёма сотрудника на работу |
| `Увольнение` | Оформление увольнения сотрудника |
| `КадровоеПеремещение` | Перевод сотрудника между отделами/должностями |
| `КадровыеПриказы` | Кадровые приказы по сотрудникам |

### Регистры

| Регистр | Назначение |
|---|---|
| `ОстаткиТоваров` | Регистр накопления остатков товаров на складе |

## Разработка

### Обновление репозитория после изменений конфигурации

1. Откройте базу в режиме **Конфигуратор**.
2. Выполните **Конфигурация → Выгрузить конфигурацию в файлы...**, указав папку репозитория — изменятся только реально изменённые файлы.
3. Зафиксируйте изменения в git:

```bash
git add .
git commit -m "Описание изменений"
git push
```

### Развёртывание конфигурации из репозитория

1. Создайте новую информационную базу в 1С (или используйте существующую).
2. В Конфигураторе выполните **Конфигурация → Загрузить конфигурацию из файлов...**, указав путь к папке репозитория.
3. Обновите конфигурацию базы данных (**F7**).
